# HOTAS Mappings ‚Äî Claude Instructions

## What this project is
A static web app (nginx in K8s via ArgoCD on Hetzner) that renders controller mapping reference cards. Two visual modes:
- **VKB view** ‚Äî overlay labels on a rendered PDF background image; fields positioned from extracted PDF form geometry (`assets/fields.json`)
- **Boopidoo view** ‚Äî CSS/HTML diagram of the Boopidoo layout (no background image needed)

The HTML viewer is embedded in `configmap.yml` as a K8s ConfigMap. Mapping data lives in `tools/*.json`.

For player hardware specs and per-rig binding constraints, see `[[project-gaming]]` in memory.

## File layout
```
tools/
  _index.json          # profile list (id + display name) ‚Äî update when adding profiles
  {profile}.json       # per-game mappings
  vkb_gladiator_{left,right}.csv  # authoritative VKB button‚Üíaction spreadsheet
  elite_dangerous_controls.csv    # ED control name reference
  prep.py              # one-shot: processes VKB PDFs ‚Üí assets/bg-*.png + assets/fields.json
assets/
  bg-lh.png / bg-rh.png   # VKB background images (generated, don't edit manually)
  fields.json             # VKB field positions (generated by prep.py)
configmap.yml            # K8s ConfigMap containing the full HTML/CSS/JS viewer
deployment.yml / ingress.yml / service.yml / namespace.yml / kustomization.yml
```

## VKB JSON schema
Each `tools/{profile}.json` has top-level `name`, `version`, and optional `notes`, plus `lh` and `rh` objects mapping VKB field IDs to display strings.

**Field ID convention:** `{cluster}_{direction}_{col}` where col `_1` = Joy_N integer, `_2` = action label. Axes use `AX_{name}`. Examples:
- `MT_A_2` ‚Äî main trigger, primary action label
- `A3_B_1` ‚Äî A3 hat down, Joy number
- `C1_E_2` ‚Äî C1 cluster push, action label
- `D1_A_2` ‚Äî D1 modifier (always `"MODIFIER"`)
- Layer-2 secondary bindings: append `(NN ACTION)` in caps to the `_2` value, e.g. `"Landing Gear (52 SHIP SPOTLIGHT)"`

The `vkb_gladiator_{left,right}.csv` files are the authoritative source for which Joy_N maps to which physical control.

## Conventions
- Prefer adding/editing JSON profile files over touching the HTML viewer unless the change requires a structural UI update
- When adding a new sim profile: add an entry to `tools/_index.json` AND create `tools/{id}.json`
- The ConfigMap HTML is large (~96KB); edit with surgical precision ‚Äî the Boopidoo and VKB sections are namespaced (`.boopidoo-view` vs `.vkb-toolbar`) and must stay independent
- Deploy: ArgoCD watches this directory; commit + push triggers sync
