► How to read mode-multiplexed keys: keypad rows 1–3 show four lines per key, one per mode. The colored letter (T / E / M / C) is the active slide-switch position. Slide to that position, then press the key.

► Slide switch is always-on: exactly one of Joy_27–30 is held at any moment. Sliding between positions briefly releases one and engages the next — no mode-multiplexed action has a plain (no-modifier) binding, so a slide-gap is harmless.

► Common bindings (active in all modes): all stick axes, both stick presses, top row, push buttons, POV hat (cardinals + diagonals), and keypad bottom row.

► POV hat diagonals = direct cockpit panel focus: ↖ Chat, ↗ Systems, ↙ Nav, ↘ Contacts. Mnemonic: ↖/↙ = left-side panels (Chat top-left, Nav left); ↗/↘ = right-side panels (Systems top-right, Contacts bottom-right).

► HeadLookToggle: Key 7 in Travel mode (modal) and Left Push (Joy_1) always-on. Off Joy_32 (hat press) — that button was firing HeadLook on every UI Select at stations. UIFocus is unbound on stick (POV diagonals direct-focus panels, making it redundant).

► Bottom-row power management: * 0 # D = WEP / SYS / ENG / Reset 2-2-2 (combat priority order left-to-right, reset on the right corner).

► Throttle vs. translation: left-stick Y is AheadThrust — direct fore/aft maneuvering thrust, spring-centered to zero thrust (not zero speed). Speed is set by the 0% / 50% / 100% buttons; the Y axis gives a delta on top of that. Both coexist. Left twist is VerticalThrustRaw (heave). Axes match VKB joystick layout: stick Y = surge, twist = heave.

► Pitch axis is inverted (pull back = nose up).

► Diagonal hat encoding: ED uses a two-key combo internally (primary cardinal + Modifier child element pointing to the second cardinal). Not a single Joy_POV1UpLeft key — that identifier does not exist in ED’s binding schema.
